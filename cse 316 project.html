Theory & Explanations

Learn about each disk scheduling algorithm.

FCFS (First Come First Serve)

Visits requests strictly in the order they arrived.


  • Advantages: Simple to implement, no starvation.
  • Disadvantages: Total seek time is high; wild swings from one end to another.

SSTF (Shortest Seek Time First)

Picks the request closest to the current head position.


  • Advantages: Drastically reduces average seek time.
  • Disadvantages: Can cause starvation for requests far away.

SCAN (Elevator Algorithm)

Head moves in one direction treating requests, then reverses direction at the boundary.


  • Advantages: Good throughput, bounded waiting time.
  • Disadvantages: Head moves to extreme ends even if no requests are there.

C-SCAN (Circular SCAN)

Head moves in one direction, treating requests. When it hits the boundary, it jumps to the other end without reading, and continues.


  • Advantages: Provides a more uniform wait time than SCAN.
  • Disadvantages: The jump adds to the seek time.

About Project

CSE 316 Operating Systems Project

Project Details


Title: Advanced Disk Scheduling Simulator

Description: A visually interactive tool to determine and analyze different OS disk scheduling algorithms.


Tech Stack: HTML5, CSS3 Variables, ES6 JavaScript, Chart.js


Summary / Recommendation Rule of Thumb:

- If the disk load is light, FCFS is acceptable.
- SSTF is commonly used but can starve outer tracks under heavy load.
- SCAN and C-SCAN are best for heavy load because they prevent starvation and reduce variance in wait times.
- C-SCAN yields the most uniform wait time across all tracks.
